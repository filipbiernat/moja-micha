# TASK-017 Plan — v2.2 Dietitian Daily Summary

## Goal

Implement GamePlan v2.2: replace the legacy plain-text daily AI insight with a stored Markdown daily summary generated by a dietitian-style prompt, enriched with user profile and 7-day history context.

## Scope

GamePlan v2.2 checklist items:

- [ ] Add new settings fields: sex, age, diet goal
- [ ] Add PL/EN translations for the new settings and daily summary UI
- [ ] Add DB migration for `daily_summaries`
- [ ] Add DB layer: `upsertDailySummary`, `getDailySummary`
- [ ] Replace `getDailyInsight` with `getDailySummary` in `services/openai.ts`
- [ ] Add helper query for 7-day average calories
- [ ] Install `react-native-markdown-display`
- [ ] Add `DailySummaryCard` component with empty/loading/error/content states
- [ ] Replace the old "Ask AI" block in `DayView`
- [ ] Handle days with missing calorie data via `caloriesAvailable`

## Architecture Decisions

- **Persist summaries in SQLite** — a generated summary is cached per day and overwritten on refresh or auto-regeneration.
- **Markdown-first output** — the AI service returns Markdown only; the UI renders it through `react-native-markdown-display`.
- **Whole-day context** — the prompt includes meals list, total kcal, calorie-goal delta when possible, regularity context, profile settings, and rolling 7-day average.
- **Date-wide availability** — summaries can be generated for any browsed day, not only today.
- **Safe async updates** — keep the generation-ref pattern already proven in `DayView` to avoid stale state writes when changing dates.
- **Light auto-refresh** — when a meal is added or edited for a late-day slot on today, regenerate the stored summary asynchronously after save.

## Implementation Phases

### Phase 1: Data + settings model

1. Extend settings keys/types for sex, age, and diet goal.
2. Add `daily_summaries` schema and DB accessors.
3. Add a meals helper for rolling 7-day calorie average.
4. Create and wire the Drizzle migration.

### Phase 2: AI summary generation

1. Remove the legacy `getDailyInsight()` API.
2. Add a typed `getDailySummary()` API that returns Markdown.
3. Build prompt context for:
    - meals of the selected day
    - calorie total and goal delta when available
    - `caloriesAvailable`
    - meal count and times
    - user profile settings
    - last-7-days average kcal

### Phase 3: UI integration

1. Add settings UI for user profile fields.
2. Add `DailySummaryCard` with generate / refresh / loading / error / content states.
3. Replace the old `Ask AI` block in `DayView`.
4. Keep the component available for any browsed date.

### Phase 4: Save-flow integration + docs

1. Trigger async late-day auto-regeneration after relevant meal saves.
2. Add/update i18n strings and install markdown dependency.
3. Check off completed GamePlan v2.2 items.
4. Run review, tests, and finalize task docs.
